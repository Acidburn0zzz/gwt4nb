GWT4NB under the hood

This page explains the low-level integration mechanisms that glue GWT Toolkit with the NetBeans build system, it is meants for our future contributors in particular.

Enhancing a Web Project to use GWT4NB has the following consequences:
  • The main build file of the project is patched to include the nbproject/build-gwt.xml file
  • The following files are created in the nbproject directory:
    • gwt.properties
    • build-gwt.xml
gwt.properties
stores GWT-specific project data. As of GWT4NB 1.2 it is only the name of the GWT module that is passed to GWTCompile
build-gwt.xml
Enhances the ant targets of the main project build file. Upon deployment GWT java code is compiled into JavaScript and copies the output to the depoyment directory. When debugging the debugger connects to both: application server and GWTShell.
Note that the embedded Tomcat is always disabled when using GWT4NB. This is necessary to make sure the other features of NetBeans Web project work as expected.
